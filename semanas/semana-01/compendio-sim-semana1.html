1. Definición e Importancia de la Simulación

1.1 Definición

La simulación es una técnica poderosa utilizada para el análisis y estudio de sistemas complejos.

Origen Histórico

Su origen se remonta a la aplicación de la probabilidad y estadística a mediados de la Segunda Guerra Mundial, conocida como el método Monte Carlo, para resolver problemas demasiado costosos o complicados para ser tratados analíticamente o experimentalmente (Prawda, 2004).

Definiciones Formales

Shannon

"La simulación es el proceso de diseñar un modelo de un sistema real y llevar a término experiencias con él, con la finalidad de comprender el comportamiento del sistema o evaluar nuevas estrategias, dentro de los límites impuestos por un cierto criterio o un conjunto de ellos, para el funcionamiento del sistema" (Shannon, 1988).

Gordon (1991)

"La simulación se define como la técnica de resolver problemas siguiendo los cambios en el tiempo de un modelo dinámico de un sistema" (Gordon, 1991).

1.2 Importancia y Propósito

La simulación es fundamental en diversas disciplinas:

Ingeniería Industrial
Investigación de Operaciones
Economía
Medicina

Capacidades de la Simulación

Evaluar alternativas sin riesgo

Permite "probar los cambios de diseño de ingeniería sin interrumpir el sistema que se está modelando" (Banks et al., 2005).

Responder preguntas estratégicas

Se utiliza para responder preguntas del tipo "¿qué sucede si?" (what-if)

Análisis de sistemas complejos

Esencial para sistemas demasiado complejos para un análisis analítico satisfactorio

2. Modelo, Sistema y Evento

2.1 Sistema

Un sistema se refiere a un conjunto de elementos interrelacionados que interactúan entre sí para lograr un objetivo común.

Un sistema se caracteriza por tener cuatro componentes fundamentales...

Característica 1: Elementos

Elementos

Los componentes básicos que conforman el sistema

En simulación: Se denominan entidades

Ejemplos: Clientes en un banco, productos en una línea de producción, pacientes en un hospital

Característica 2: Relaciones

Relaciones

Las conexiones e interacciones entre los elementos del sistema

Tipos de relaciones:

  • Dependencias funcionales
  • Flujos de información
  • Jerarquías organizacionales
  • Restricciones operativas

Característica 3: Entradas y Salidas

Entradas y Salidas

Los flujos de recursos que ingresan y salen del sistema

Entradas (Inputs)
  • Materias primas
  • Información
  • Energía
  • Personal
Salidas (Outputs)
  • Productos terminados
  • Servicios
  • Reportes
  • Residuos

Característica 4: Propiedades

Propiedades

Las características y atributos que definen el comportamiento del sistema

Capacidad: Límites operativos del sistema
Eficiencia: Relación entre recursos y resultados
Confiabilidad: Consistencia en el desempeño

Resumen: Características del Sistema

Un sistema completo integra todas sus características

Flujos del sistema

Propiedades

Punto Clave: En simulación, estos componentes se modelan para replicar el comportamiento del sistema real

2.2 Modelo

El modelo es una representación simplificada y abstracta de un sistema real que se utiliza para comprender y predecir su comportamiento.

Características de un Buen Modelo:

  • Describir todas las propiedades importantes del sistema
  • Ser fácil de entender y modificar
  • Perseguir objetivos claros

2.3 Evento

Un evento es un acontecimiento que sucede en un instante de tiempo determinado en la simulación, lo que provoca un cambio en el estado actual del sistema.

3. Tipos de Sistemas y Modelos

3.1 Clasificación por Representación

Los modelos se clasifican según su estructura y forma de representación del sistema real

Tres tipos principales de modelos según su representación

1. Modelos Icónicos (Físicos)

Modelos Icónicos

Representaciones físicas a escala del sistema real

Características:
  • Representación física tangible
  • Escala reducida o ampliada
  • Mantienen proporciones del original
  • Permiten visualización directa
Ejemplos:
  • Maquetas arquitectónicas
  • Prototipos de productos
  • Plantas piloto industriales
  • Modelos de aviones a escala

Ejemplos de modelos icónicos: maquetas y prototipos físicos

2. Modelos Analógicos

Ejemplos de modelos analógicos: mapas y diagramas

Modelos Analógicos

Utilizan analogías para representar el sistema

Características:
  • No reflejan estructura detallada
  • Representan relaciones entre partes
  • Usan símbolos y convenciones
  • Facilitan comprensión conceptual
Ejemplos:
  • Mapas geográficos
  • Diagramas de circuitos eléctricos
  • Organigramas organizacionales
  • Diagramas de flujo de procesos

3. Modelos Simbólicos (Matemáticos)

Modelos Simbólicos

Codifican características mediante lenguaje matemático

Características:
  • Usan símbolos matemáticos
  • Expresan relaciones cuantitativas
  • Permiten análisis riguroso
  • Base para simulación computacional
Ejemplos:
  • Ecuaciones diferenciales
  • Modelos estadísticos
  • Algoritmos de optimización
  • Modelos de simulación computacional

Ejemplos de modelos simbólicos: ecuaciones y algoritmos

Fundamentales para simulación: Son la base de todos los modelos computacionales

Comparación de Tipos de Modelos

Cada tipo de modelo tiene sus propias ventajas y aplicaciones

Icónicos

Ventaja: Visualización directa

Uso: Prototipos y maquetas

Analógicos

Ventaja: Comprensión conceptual

Uso: Diagramas y mapas

Simbólicos

Ventaja: Análisis cuantitativo

Uso: Simulación computacional

En simulación: Los modelos simbólicos son los más utilizados por su capacidad de procesamiento computacional

3.2 Clasificación por Naturaleza

Característica Modelo Discreto Modelo Continuo
Cambio del sistema Solo cambia en instantes específicos (eventos) Cambia continuamente a lo largo del tiempo
Tiempo El reloj salta de un punto discreto a otro Modelado por ecuaciones diferenciales
Unidad de modelado Modela elementos individuales (entidades) Modela flujos
Ejemplos Peaje, cajero automático, líneas de espera Flujo de líquido, transferencia de calor

3.3 Clasificación por Aleatoriedad

Deterministas

No interviene el azar. Bajo las mismas condiciones iniciales se obtienen siempre los mismos resultados.

Prácticamente no existen en la naturaleza
Estocásticos

Interviene el azar o elementos que no se controlan. Utilizan variables aleatorias mediante distribuciones de probabilidad.

Simulación de Monte Carlo

Modelo estático que no interviene el tiempo. Se basa en la aleatoriedad usando números aleatorios para obtener un resumen estadístico de las observaciones simuladas.

3.4 Estrategias de Simulación Discreta

1. Programación de Eventos

Tratamiento secuencial de acontecimientos ordenados por tiempo en una lista de eventos

2. Interacción de Procesos

Se enfoca en la experiencia de la entidad a través del sistema. Usan interfaz gráfica (Arena, ProModel)

3. Examinación de Actividades

Define condiciones para empezar y finalizar cada actividad. El tiempo avanza en incrementos iguales

Caso 2: Sistemas de Manufactura con Máquinas de Ensamblaje

Manufactura con Múltiples Bucles

Descripción: Estudio de sistemas de manufactura con máquinas de ensamblaje/desensamblaje, múltiples bucles y diseño general. Se evalúa el desempeño de producción mediante variables como velocidad de producción, nivel de inventario y flujo de materiales.

Características del Sistema
  • Variables principales: Velocidad de producción, nivel de inventario, flujo de materiales
  • Comportamiento temporal: Las variables evolucionan a lo largo del tiempo
  • Modelado matemático: Utiliza ecuaciones diferenciales para representar el sistema
  • Enfoque: Se centra en flujos agregados más que en entidades individuales

Referencia: Scrivano, S. (2023). Continuous-flow simulation of manufacturing systems with assembly/disassembly machines, multiple loops and general layout. Journal of Manufacturing Systems.

Caso 3: Interacciones Tren-Peatón

Características del Sistema

  • Flujo de peatones: Circulación en áreas urbanas
  • Eventos de intersección: Cruces, semáforos, tiempos de espera
  • Interacciones: Los eventos puntuales afectan los patrones de movimiento

Caso 4: Sistema de Pensión para Docentes

Descripción: Modelo de optimización para un sistema de pensión de contribución definida para docentes. Se optimizan las contribuciones considerando tasas fijas, períodos de servicio y salarios base establecidos.

Características del Sistema
  • Variables controladas: Tasas de contribución, períodos de servicio, salarios base
  • Parámetros fijos: Todas las variables tienen valores establecidos

Caso 5: Modelado de COVID-19 en Argelia

Difusión Epidemiológica con Perturbaciones

Descripción: Estudio comparativo de enfoques de modelado computacional para analizar la difusión de COVID-19 en Argelia. Utiliza un modelo compartimental con perturbaciones aleatorias (ruido blanco) para capturar la incertidumbre epidemiológica.

Características del Sistema
  • Tasas variables: Contagio, incubación y recuperación con variabilidad
  • Perturbaciones aleatorias: Ruido blanco que afecta la propagación
  • Múltiples ejecuciones: Se requieren varias corridas para obtener resultados confiables
  • Incertidumbre: Los resultados varían entre diferentes simulaciones

Referencia: Kadri, A., et al. (2025). A comparative study of deterministic and stochastic computational modeling approaches for analyzing and optimizing COVID-19 control. Scientific Reports.

Caso 6: Simulación Integral en Salud

  • Necesidad de combinar técnicas de modelado

  1. ¿El sistema tiene eventos puntuales Y procesos continuos?
  2. ¿Los componentes discretos afectan los flujos continuos?

5. Ventajas y Limitaciones

5.1 Ventajas de la Simulación

La simulación es una herramienta flexible, poderosa e intuitiva que ofrece múltiples beneficios organizados en categorías clave:

Análisis y Comprensión
Experimentación y Riesgo
Costo y Tiempo
Comunicación

Análisis y Comprensión

Análisis y Comprensión

  • Mejora el proceso de decisión al permitir el análisis de efectos de cambios internos o externos
  • Facilita la comprensión de la operación de sistemas complejos
  • Detecta variables más importantes y entiende su relación
  • Identifica cuellos de botella y puntos críticos del sistema

Experimentación y Riesgo

Experimentación y Riesgo

  • Permite experimentar con nuevas situaciones (sistemas nuevos o existentes con poca información)
  • Anticipa resultados no previstos antes de la implementación real
  • Prueba diferentes alternativas y diseños sin incurrir en los costos del sistema real
  • Reduce riesgos de inversión y toma de decisiones

Costo y Tiempo

Costo y Tiempo

  • Experimentar con un modelo suele ser más económico que con un sistema real
  • Simula años de operación en cuestión de segundos o minutos
  • Genera observaciones estadísticas de manera rápida y eficiente
  • Reduce costos de implementación al validar conceptos previamente

Flexibilidad y Precisión

Flexibilidad y Precisión

  • El modelo puede ser muy detallado y preciso - los modelos de simulación tienen pocas restricciones comparados con los analíticos
  • Control y manipulación de variables de entrada con mayor precisión que en un sistema real
  • Adaptabilidad a diferentes escenarios y condiciones operativas
  • Escalabilidad para modelar desde componentes hasta sistemas completos

Comunicación

Comunicación

  • Uso de animaciones (2D, 3D) permite "comunicar la esencia de un modelo de simulación... a los administradores y otras personas clave" (Law & Kelton, 2000)
  • Visualización intuitiva de procesos complejos
  • Facilita el consenso entre stakeholders y equipos multidisciplinarios
  • Herramienta de presentación efectiva para justificar inversiones y cambios

5.2 Limitaciones de la Simulación

A pesar de sus beneficios, la simulación presenta inconvenientes que deben considerarse cuidadosamente:

No Optimización
Costo y Tiempo
Resultados Estadísticos
Dependencias

No es Técnica de Optimización

No es Técnica de Optimización

  • Solo analiza alternativas y proporciona estimaciones estadísticas
  • No proporciona resultados exactos u óptimos (a menos que se utilicen paquetes especializados como OptQuest)
  • Requiere definir escenarios a evaluar previamente
  • No garantiza encontrar la mejor solución posible automáticamente

Costo y Tiempo de Desarrollo

Costo y Tiempo de Desarrollo

  • Costosa en horas de desarrollo y recursos humanos especializados
  • Requiere tiempo considerable para la verificación del programa y validación de resultados
  • Los estudios pueden requerir meses para modelar y simular sistemas complejos
  • Inversión inicial alta en software, capacitación y desarrollo

Resultados Estadísticos

Resultados Estadísticos

  • Proporciona solo estimaciones estadísticas, no resultados exactos
  • Análisis de sensibilidad costoso - requiere múltiples corridas con diferentes valores
  • Intervalos de confianza necesarios para interpretar resultados
  • Variabilidad inherente en los resultados debido a la aleatoriedad

Imitación Parcial

Imitación Parcial

  • "La simulación imita, pero no reproduce exactamente (en un 100%) la vida" (Shannon, 1988)
  • Algunos aspectos de la realidad son difíciles o imposibles de simular
  • Simplificaciones necesarias pueden omitir factores importantes
  • Comportamiento humano impredecible difícil de modelar completamente

Dependencia del Software y Personal

Dependencia del Software y Personal

  • Limitada por la habilidad del programador y del equipo de modelado
  • Existencia de software especializado puede ser limitante o costosa
  • Conocimiento amplio requerido en software, probabilidad y estadística
  • Interpretación correcta de resultados requiere experiencia especializada

Definición de Simulación

¿Cómo define Shannon la simulación?

Proceso de Diseño y Experimentación

Proceso de diseñar un modelo de un sistema real y llevar a término experiencias con él, para comprender el comportamiento del sistema o evaluar nuevas estrategias.

Base fundamental de la simulación

Modelo vs Sistema

¿Cuál es la diferencia entre modelo y sistema?

Representación vs Realidad

Sistema: conjunto de elementos interrelacionados con objetivo común. Modelo: representación simplificada y abstracta del sistema real.

El modelo imita el sistema

Tipos de Modelos

¿Cuáles son los tres tipos de modelos por representación?

Icónicos, Analógicos, Simbólicos

Icónicos: maquetas a escala, Analógicos: mapas y diagramas, Simbólicos: matemáticos para simulación computacional.

Del físico al matemático

Discreto vs Continuo

¿Cuál es la diferencia entre sistemas discretos y continuos?

Cambios por Eventos vs Flujo

Discreto: cambia solo en instantes específicos (eventos). Continuo: cambia continuamente con ecuaciones diferenciales.

Ejemplos: peaje vs flujo de líquido

Conclusión

La simulación es una herramienta fundamental para el análisis de sistemas complejos que permite:

  • Comprender el comportamiento de sistemas reales
  • Evaluar alternativas sin riesgo
  • Optimizar procesos y reducir costos
  • Tomar decisiones informadas basadas en evidencia
