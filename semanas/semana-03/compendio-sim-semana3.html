1. Números Pseudoaleatorios Uniformes

1.1 Definición y Naturaleza

Los números pseudoaleatorios son esenciales en la simulación estocástica, ya que permiten reproducir el comportamiento de variables aleatorias con cualquier función de probabilidad.

🎯 ¿Por qué "Pseudoaleatorios"?

Se denominan "pseudoaleatorios" (falso aleatorios) porque, aunque tienen la apariencia de ser aleatorios, en realidad son generados a partir de un algoritmo matemático determinista.

Definición Matemática

Un número pseudoaleatorio (\(R_i\) o \(U_i\)) es el valor que toma una variable aleatoria \(x\) que posee una distribución de probabilidad uniforme en el intervalo \([0, 1]\).

Rango estándar: \(U_i \sim \mathcal{U}(0, 1)\)

¿Por qué [0, 1]? Este rango es el más común porque simplifica las transformaciones a otras distribuciones.

Requisito clave: Para que una secuencia de números pseudoaleatorios sea útil en simulación, debe cumplir las mismas propiedades estadísticas relevantes que una secuencia verdaderamente aleatoria \(\mathcal{U}(0, 1)\).

1.2 Propiedades de Uniformidad e Independencia

La secuencia de números pseudoaleatorios generada debe satisfacer dos propiedades principales, las cuales deben ser validadas mediante pruebas estadísticas:

Propiedad 1: Uniformidad

📊 Distribución U(0,1)

La secuencia debe ajustarse a una distribución \(\mathcal{U}(0, 1)\).

Función de Densidad
\(f(x) = \begin{cases} 1 & \text{si } x \in (0, 1) \\ 0 & \text{si } x \notin (0, 1) \end{cases}\)
Parámetros Teóricos
  • Media: \(\mu = 0.5\)
  • Varianza: \(\sigma^2 = \frac{1}{12}\)
  • Desv. Estándar: \(\sigma \approx 0.2887\)

Propiedad 2: Independencia

🔗 Independencia Estadística

Los números deben ser estadísticamente independientes, sin ninguna correlación aparente entre ellos, actuando como valores uniformemente dispersos.

Características Clave
  • Sin correlación: No debe existir patrón predecible entre números consecutivos
  • Independencia entre réplicas: Clave en la simulación
  • Dispersión uniforme: Los valores deben estar uniformemente distribuidos

Requisitos adicionales: Un generador eficiente debe tener un periodo suficientemente grande y ser reproducible (arrancando con las mismas condiciones iniciales, debe generar siempre la misma secuencia).

2. Generador Congruencial Lineal Mixto

Introducción al GCL

El Generador Congruencial Lineal (GCL) es uno de los algoritmos más antiguos, conocidos y utilizados para generar números pseudoaleatorios.

🔄 Características

  • Algoritmo recursivo: Produce una secuencia de números pseudoaleatorios
  • Determinista: Misma semilla = misma secuencia
  • Eficiente: Computacionalmente rápido
  • Ampliamente usado: Base de muchos generadores modernos

2.1 Definición del Generador Mixto

El GCL Mixto se define por la siguiente relación de recurrencia, la cual genera una secuencia de números enteros \(X_i\):

Fórmula recursiva:

\[X_{i+1} = (a \cdot X_i + c) \bmod m \quad \text{para } i = 0, 1, 2, 3, \ldots\]

Clasificación:

  • El generador es Mixto si \(c \neq 0\)
  • Si \(c = 0\), se denomina Congruencial Multiplicativo

Parámetros del GCL

Los parámetros enteros deben ser mayores que cero:

Parámetro Símbolo Descripción Restricción
Semilla \(X_0\) Valor inicial necesario para empezar el algoritmo \(X_0 > 0\)
Multiplicador \(a\) Constante multiplicativa \(a > 0\)
Incremento \(c\) Constante aditiva (avance) \(c \geq 0\)
Módulo \(m\) Máximo número posible de valores distintos \(m > 0\)

Rango de salida: \(X_{i+1} \in \{0, 1, 2, \ldots, m-1\}\)

2.2 Obtención de Números Uniformes

La ecuación recursiva genera enteros \(X_i \in \{0, 1, \ldots, m-1\}\). Para transformar la secuencia a números pseudoaleatorios \(U_i\) en el intervalo \((0, 1)\), se utiliza:

Transformación principal:

\[U_i = \frac{X_i}{m}\]

Alternativa: En algunos casos se utiliza \(R_i = \frac{X_i}{m-1}\) para obtener valores entre 0 y 1.

Nota: La primera transformación es más común y estándar en la literatura.

2.3 Periodo Máximo

🔄 Ciclo de Vida del Generador

El periodo es la longitud de la secuencia antes de que se repita cíclicamente.

El periodo máximo que puede alcanzar un GCL es igual a su módulo: \(p = m\)

🎯 Teorema de Hull-Dobell

Para que un generador congruencial lineal tenga periodo completo (\(p = m\)) para cualquier semilla, debe cumplir las tres condiciones:

  1. \(c\) y \(m\) son primos relativos: \(\gcd(c, m) = 1\)
  2. \(a - 1\) es múltiplo de todos los factores primos de \(m\)
  3. Si \(m\) es múltiplo de 4, entonces \(a - 1\) también lo es

3. Pruebas Básicas de Uniformidad

Propósito de las Pruebas

La finalidad de los tests empíricos de uniformidad es garantizar que la secuencia de números generados se ajusta a una distribución \(\mathcal{U}(0, 1)\).

🎯 Importancia

  • Validación estadística: Confirmar propiedades teóricas
  • Calidad del generador: Detectar deficiencias
  • Confiabilidad: Asegurar resultados de simulación válidos

3.1 Prueba de Kolmogorov-Smirnov (K-S)

📊 Test No Paramétrico de Bondad de Ajuste

Propósito

Compara la función de distribución empírica de los datos generados con la función de distribución acumulada de la distribución uniforme teórica.

Aplicación

Es útil para determinar si los números se distribuyen uniformemente. Se aplica a distribuciones teóricas continuas.

🔍 Procedimiento Básico
  1. Ordenar los números pseudoaleatorios
  2. Calcular la desviación máxima \(D\) (máxima diferencia absoluta entre distribución teórica y observada)
  3. Comparar con valor crítico \(d_{\alpha, n}\)
  4. Si \(D > d_{\alpha, n}\), se rechaza \(H_0\) (uniformidad)

3.2 Prueba Chi-Cuadrado (\(\chi^2\))

📈 Prueba de Pearson o Prueba de Frecuencias

Propósito

Mide la discrepancia entre la frecuencia observada (\(F_o\)) de los números en ciertos intervalos y la frecuencia esperada (\(F_e\)) si los números fueran perfectamente uniformes.

Principio: En una distribución uniforme, se espera que el número de datos en cada intervalo sea aproximadamente igual.

Procedimiento Chi-Cuadrado

🔢 Pasos del Procedimiento

  1. Agrupar: Los \(n\) números aleatorios generados en \(K\) clases disjuntas de igual amplitud \(A\)
  2. Calcular frecuencias:
    • Frecuencia observada: \(F_o\)
    • Frecuencia esperada: \(F_e = n \cdot A_i\)
  3. Calcular estadístico: \(\chi^2\)
  4. Comparar: Con valor crítico \(\chi^2_{\alpha, K-1}\)

Decisión: Si \(\chi^2 > \chi^2_{\alpha, K-1}\), se rechaza \(H_0\) (uniformidad)

3.3 Pruebas de Medias y Varianzas

📊 Pruebas de Hipótesis Paramétricas

Estas pruebas validan si las propiedades muestrales coinciden con los parámetros teóricos de la distribución \(\mathcal{U}(0, 1)\).

🎯 Prueba de Medias

Contrasta la hipótesis nula de que la media poblacional es \(\frac{1}{2}\):

\(H_0: \mu = \frac{1}{2}\)

Si hay gran cantidad de datos uniformes entre 0 y 1, se esperaría que el promedio fuera 0.5

📐 Prueba de Varianzas

Contrasta la hipótesis nula de que la varianza poblacional es \(\frac{1}{12}\):

\(H_0: \sigma^2 = \frac{1}{12}\)

De acuerdo con la teoría estadística de la distribución uniforme

4. Implementación Básica

4.1 Implementación en Excel

Excel es una herramienta fundamental para la implementación manual y la simulación básica.

Generación de Números Uniformes en Excel

🎲 Función Incorporada

Función básica:
=ALEATORIO() o =RAND()

Características:

  • Genera un número pseudoaleatorio entre 0 y 1
  • Es una función "volátil" que se actualiza cada vez que se realiza un cálculo
  • Se puede forzar actualización con F9
🔄 Variables Uniformes X(a,b)

Para generar una variable aleatoria \(X\) uniformemente distribuida en \([a, b]\):

\(X = a + (b - a) \cdot R_i\)
Ejemplo Excel (a=245, b=255):
=245+10*ALEATORIO()

Implementación del GCL en Excel

⚙️ Fórmula Recursiva

La relación \(X_{i+1} = (a \cdot X_i + c) \bmod m\) se puede implementar utilizando una fórmula de celda que calcula el residuo (módulo).

🧮 Cálculo del Residuo

Fórmula general para calcular \(X_{i+1}\) en Excel:

Fórmula completa:
=(D$3*C10+D$4)/D$5-ENTERO((D$3*C10+D$4)/D$5))*D$5

Donde: D$3=a, D$4=c, D$5=m, C10=\(X_i\) anterior

📊 Número Pseudoaleatorio Final

El número pseudoaleatorio final \(U_i\) se obtiene:

\(U_i = \frac{X_i}{m}\)

4.2 Implementación en R

El lenguaje R es una implementación del lenguaje S y es adecuado para la simulación estadística.

Generación de Números Uniformes en R

📊 Funciones Básicas

Función principal:
runif(n) # Genera n valores de U(0,1)
🌱 Control de Semilla
set.seed(123) # Fijar semilla para reproducibilidad
runif(10) # Generar 10 números uniformes
🔄 Variables Uniformes X(a,b)

Para simular \(\mathcal{U}[a, b]\):

# Método 1: Transformación manual
U <- runif(n)
X <- a + U * (b - a)

# Método 2: Función directa
runif(n, min = a, max = b)

Implementación del GCL en R

🔧 Generadores en R

Generador por defecto: R emplea por defecto el generador Mersenne-Twister.

⚙️ Implementación Manual del GCL
# Función GCL personalizada
lcg <- function(n, a, c, m, seed) {
X <- numeric(n)
X[1] <- seed
for(i in 2:n) {
X[i] <- (a * X[i-1] + c) %% m
}
return(X / m) # Convertir a U(0,1)
}
📦 Paquetes Especializados

Los GCL también se pueden implementar utilizando paquetes como simres, que ofrece la función rlcg().

Ejemplo Práctico

Implementación Paso a Paso

🎯 Ejemplo: GCL con parámetros específicos

Implementemos un GCL con los siguientes parámetros:

Semilla (X₀) 1
Multiplicador (a) 5
Incremento (c) 3
Módulo (m) 8

Cálculos Manuales

🧮 Secuencia Generada

X₁ = (5 × 1 + 3) mod 8 = 8 mod 8 = 0 → U₁ = 0/8 = 0.000
X₂ = (5 × 0 + 3) mod 8 = 3 mod 8 = 3 → U₂ = 3/8 = 0.375
X₃ = (5 × 3 + 3) mod 8 = 18 mod 8 = 2 → U₃ = 2/8 = 0.250
X₄ = (5 × 2 + 3) mod 8 = 13 mod 8 = 5 → U₄ = 5/8 = 0.625
X₅ = (5 × 5 + 3) mod 8 = 28 mod 8 = 4 → U₅ = 4/8 = 0.500
X₆ = (5 × 4 + 3) mod 8 = 23 mod 8 = 7 → U₆ = 7/8 = 0.875
X₇ = (5 × 7 + 3) mod 8 = 38 mod 8 = 6 → U₇ = 6/8 = 0.750
X₈ = (5 × 6 + 3) mod 8 = 33 mod 8 = 1 → U₈ = 1/8 = 0.125
X₉ = (5 × 1 + 3) mod 8 = 8 mod 8 = 0 → ¡Se repite!

Observación: El periodo es 8, que coincide con el módulo m = 8. ¡Periodo completo!
